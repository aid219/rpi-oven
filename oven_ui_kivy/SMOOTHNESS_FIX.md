# 🎯 Решение проблемы с плавностью интерфейса

## Проблема
- FPS высокий (70-80)
- Но движение/скролл **не плавный** (видно рывки)

## Причина
1. **Tearing** — разрывы кадров без vsync
2. **Неравномерный frametime** — кадры приходят не равномерно
3. **Частая перерисовка canvas** — индикаторы обновляются каждый кадр

---

## ✅ Что исправлено

### 1. main_optimized.py

#### Включён vsync
```python
Config.set('graphics', 'vsync', '1')  # Синхронизация с частотой дисплея
Config.set('graphics', 'maxfps', '0')  # Без искусственного ограничения
```

#### Увеличена длительность перехода
```python
self.sm.transition = SlideTransition(duration=0.35)  # Более плавный переход
```

#### Плавная анимация
```python
Clock._max_fps = 0  # Без искусственного ограничения
```

### 2. dashboard_screen.py

#### Оптимизирован CircularProgress
```python
# Было: обновление каждый кадр
self.bind(size=self._update, pos=self._update)

# Стало: обновление 20 FPS (каждые 0.05 сек)
self._schedule_update()  # Отложенное обновление
```

**Результат:** Меньше нагрузка на CPU/GPU, плавнее анимация.

## 🔧 Если всё ещё не плавно

### 1. Проверьте частоту обновления дисплея
```bash
# На Raspberry Pi
tvservice -s
```

Должно быть **60Hz**. Если меньше — дисплей не поддерживает.

### 2. Отключите композитинг (если есть)
```bash
# Для X11
xcompmgr -c &
```

### 3. Уменьшите сложность UI
- Меньше анимаций
- Меньше виджетов с canvas
- Используйте текстуры вместо рисования

### 4. Проверьте температуру CPU
```bash
vcgencmd measure_temp
```

Если >80°C — троттлинг снижает производительность.
