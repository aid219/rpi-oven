# 🚀 Как получить 24+ FPS

## 📊 Текущая ситуация

**Сейчас:** 16.6 FPS  
**Цель:** 24+ FPS

---

## 🔧 Способ 1: Отключить VSYNC (быстро)

### Запустить версию без VSYNC:
```bash
cd /home/qummy/rpi-oven/oven_ui_kivy
python3 main_framebuffer.py
```

**Что делает:**
- `Config.set('graphics', 'vsync', '0')` — отключает синхронизацию
- Может дать **20-25 FPS**
- **Минус:** возможен tearing (разрывы)

---

## 🔧 Способ 2: Оптимизация config.txt (сложно)

### Запустить скрипт оптимизации:
```bash
cd /home/qummy/rpi-oven/oven_ui_kivy
sudo bash optimize_config.sh
sudo reboot
```

**Что меняет:**
- `core_freq=500` — разгон ядра
- `core_freq_turbo=550` — турбо режим
- `gpu_freq=300` — частота GPU
- `gpu_mem=128` — больше памяти GPU
- `cma-16` — меньше памяти под DMA

**Ожидаемый результат:** +2-4 FPS

---

## 🔧 Способ 3: Прямой framebuffer (максимум)

### Использовать Pygame + прямую запись:

```python
# Вернуться к fb_animation.py
python3 ~/fb_animation.py
```

**Почему быстрее:**
- Нет накладных расходов OpenGL
- Нет DRM/KMS посредников
- Прямая запись в `/dev/fb0`

**Ожидаемый результат:** 18-22 FPS

---

## 🔧 Способ 4: Уменьшить разрешение (радикально)

### Уменьшить количество пикселей:

```python
# main_framebuffer.py
Config.set('graphics', 'width', '240')   # Вместо 320
Config.set('graphics', 'height', '360')  # Вместо 480
```

**Почему быстрее:**
- Меньше пикселей = быстрее передача
- 240×360 = 86,400 пикселей (вместо 153,600)
- **−44% данных** = **+40% скорости**

**Ожидаемый результат:** 22-28 FPS

## 🎯 Рекомендация

### Попробовать по порядку:

1. **Запустить `main_framebuffer.py`** — проверить VSYNC off
2. **Запустить `optimize_config.sh` + reboot** — разгон
3. **Если мало — уменьшить разрешение** до 240×360

### Комбо для максимума:
```bash
# 1. Оптимизировать config
sudo bash optimize_config.sh
sudo reboot

# 2. Запустить с отключенным VSYNC
python3 main_framebuffer.py

# 3. Если всё ещё мало — уменьшить разрешение в коде
```

## ⚠️ Предупреждения

1. **Отключение VSYNC** — возможны разрывы изображения
2. **Разгон (core_freq)** — может вызвать нестабильность
3. **Уменьшение разрешения** — хуже качество

**Делайте бэкап config.txt перед изменениями!**
