## ✅ Выполнено

### 1. Настройка тачскрина FT6336U

**Проблема:** Стандартный overlay `ft6336u` не работает — не настраивает GPIO interrupt.

**Решение:**
- Создан кастомный device tree overlay `/boot/firmware/overlays/ft6336u-fix.dtbo`
- Настроен interrupt через GPIO 27
- Драйвер: `edt-ft5x06` с совместимостью `focaltech,ft6236`

**Файлы:**
- `/boot/firmware/overlays/ft6336u-fix.dtbo` — кастомный overlay
- `/boot/firmware/config.txt` — добавлено `dtoverlay=ft6336u-fix`
- `/etc/modules-load.d/touchscreen.conf` — автозагрузка модулей
- `FT6336U_TOUCHSCREEN_SETUP.md` — полная инструкция по настройке

**Проверка:**
- Тачскрин определяется как `/dev/input/eventX` (номер меняется)
- Имя устройства: `1-0038 EP0110M09`
- Работает после перезагрузки (проверено 3 раза)
- Тест: `sudo evtest /dev/input/eventX`

### 2. Pygame интерфейс для SPI дисплея

**Дисплей:**
- Подключение: SPI (mipi-dbi-spi)
- Разрешение: 320x480
- Фреймбуфер: `/dev/fb0` (16 бит, RGB565)

**Проблема с FPS:**
- KMSDRM драйвер даёт ~14-15 FPS вместо целевых 30
- Прямая запись во фреймбуфер работает медленно из-за конвертации форматов
- OpenCV показывает нормальные FPS (значит железо позволяет)

**Текущее решение:**
- Файл: `~/fb_animation.py` на Raspberry Pi
- Драйвер: `SDL_VIDEODRIVER=KMSDRM`
- Режим: `FULLSCREEN | HWSURFACE | DOUBLEBUF`

**Функционал интерфейса:**
- 4 кнопки с анимацией (тень, градиент, hover-эффект)
- Анимированная иконка с частицами
- Градиентный фон
- Отображение FPS в реальном времени
- Поддержка мыши/тача (кнопки реагируют на наведение)

**Запуск:**
```bash
python3 ~/fb_animation.py
```

**Текущий FPS: ~14-15** (ограничение KMSDRM для SPI дисплея)

## 🔧 Технические детали

### Конфигурация Raspberry Pi:
- Модель: Raspberry Pi Zero 2 W (судя по Bluetooth чипу BCM43430)
- Ядро: 6.12.47+rpt-rpi-v7
- Дисплей: panel-mipi-dbid (SPI)
- Тачскрин: FT6336U на I2C адресе 0x38

### Модули ядра:
```
edt_ft5x06     28672  0
regmap_i2c     12288  1 edt_ft5x06
```

### Ограничения:
- KMSDRM не оптимален для SPI дисплеев
- Прямая запись во фреймбуфер требует конвертации в RGB565
- Python медленный для попиксельной конвертации

## 📝 Заметки

1. **Номер input устройства меняется** при загрузке (event3, event4, event5)
   - Для стабильной работы искать по имени: `1-0038 EP0110M09`

2. **KMSDRM ограничение FPS**
   - Вероятно, связано с тем, что DRM не «родной» для SPI дисплея
   - Simple framebuffer работает, но KMSDRM использует другой путь

3. **Возможные пути улучшения FPS:**
   - Использовать C расширение для конвертации RGB565
   - Попробовать `directfb` вместо KMSDRM
   - Писать на C с прямым доступом к SPI через spidev

## ✅ Интеграция энкодера в Kivy UI

### Дата: 10 апреля 2026

**Проблема:** При попытке внедрить энкодер в Kivy приложение происходило полное зависание при запуске.

**Причина:** `pigpio.pi()` инициализировался в главном потоке Kivy, блокируя event loop.

**Решение:**
- Создан `encoder_handler.py` — обёртка для энкодера с инициализацией в отдельном потоке
- Энкодер создаётся асинхронно в `OvenApp.on_start()` через `threading.Thread`
- Опрос значений через `Clock.schedule_interval` (30 раз в секунду)
- Callback'и вызываются в главном потоке Kivy через `poll()`
- Graceful shutdown в `OvenApp.on_stop()`

**Изменённые файлы:**
| Файл | Изменения |
|------|-----------|
| `oven_ui_kivy/encoder_handler.py` | Новый — обёртка энкодера с потокобезопасной инициализацией |
| `oven_ui_kivy/main_optimized.py` | Добавлена инициализация энкодера при старте |
| `oven_ui_kivy/screens/main_wait_screen.py` | Подписка на энкодер, управление таймером вращением |
| `encoder.py` | Скопирован на Pi (уже был) |

**Функционал энкодера:**
- При вращении меняется значение таймера (секунды/минуты)
- Вправо — увеличение, влево — уменьшение
- Сброс при входе на экран
- Корректная очистка callback при уходе с экрана

**Тестирование:**
- ✅ Приложение запускается без зависания
- ✅ Энкодер инициализируется в отдельном потоке
- ✅ pigpiod запущен и настроен как systemd сервис
- ✅ Лог: `Энкодер инициализирован, начальное значение: 0`
